jsh_web项目构建

1 环境安装

1 首先安装最新的node包 v4.2.2 LTS
2 全局安装 bower gulp yo(可以不安装)

npm install -g yo bower gulp

3 gulp 的安装 除了全局 在项目目录中还需要安装一次,多版本共存的需要
npm install –save-dev gulp
4 执行 npm install 执行完成 执行 bower install
5 需要特殊处理的是 gulp-sass plugins 如果gulp-sass安装不成功,将gulp-sass 单独拿出来 复制到node_modules中

命名规范

1 文件的命名,包括样式 html 图片 js 文件夹的命名及相应的文件命名用下划线形式来命名
不要出现中横线及驼峰式命名
2 样式中类使用中横线
3 id使用下划线 建议 J_center_btn 全局变量 全部使用大写 如 BASE
4 命名使用英文 不要出现拼音

gulp

gulp知识点

什么是gulp

Gulp是一个构建系统,执行开发中的常见任务.Gulp是基于Node.js来构建的.

1.全局安装gulp

$ npm install --global gulp

2.作为项目的开发依赖安装 在对应项目中安装

$ npm install --save-dev gulp

3 项目初始化 npm init 来 创建 package.json文件

$ npm init

通过初始化来设置项目的一些信息

gulp 获取文件

  • js/app.js 精确匹配文件
  • js/*.js 仅匹配js目录下的所有后缀为.js的文件
  • js/*/.js 匹配js目录及其子目录下所有后缀为js的文件
  • !js/app.js 从匹配结果中排除js/app.js,这种方法在你想要匹配特殊文件之外的所有文件非常管用
  • *.+(js|css) 匹配根目录下所有后缀为.js或者.css的文件

gulp 的API

gulp.task() gulp.src() gulp.dest() gulp.watch()

gulp.task(name, fn)这个你应经见过了

gulp.run(tasks…)尽可能多的并行运行多个task

gulp.watch(glob, fn)当glob内容发生改变时，执行fn

gulp.src(glob)返回一个可读的stream

gulp.dest(glob)返回一个可写的stream

gulp中的顺序执行方案

在gulp对任务的定义中,可以声明任务之间的依赖. 比如
gulp.task(“sass”,[“clean”],funtion(){
//do something
})
gulp.task(“clean”,function(){
return gulp.src(‘sample.css’)
.pipe(clean())
})
在声明依赖后,可以保证任务的顺序执行. 但是要记得 return 返回流

gulp plugins

gulp-autoprefixer
根据设置浏览器版本自动处理浏览器前缀.不必考虑各浏览器兼容前缀
