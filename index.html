webpack 介绍

webpack 是前端一个工具,可以让各个模块进行加载, 预处理,再进行打包,它有grunt和gulp的所有的基本功能.

.支持CommonJs和AMD模块，意思也就是我们基本可以无痛迁移旧项目。
.支持模块加载器和插件机制，可对模块灵活定制。特别是我最爱的babel-loader，有效支持ES6。
.可以通过配置，打包成多个文件。有效利用浏览器的缓存功能提升性能。
.将样式文件和图片等静态资源也可视为模块进行打包。配合loader加载器，可以支持sass，less等CSS预处理器。
.内置有source map，即使打包在一起依旧方便调试。
.支持对css 图片等资源进行打包,从而无需借助Grunt或gulp

webpack的目标就是对项目中的静态资源进行统一整理,为产品的最终发布提供最优的打包部署方案.

安装webpack

两种方式安装webpack 一种单独下载webpack文件 另一种是用npm包的形式来安装(建议)
在全局中安装webpack
npm install webpack -g
安装完成之后,就有了全局的webpack命令. 直接执行此命令会默认使用当前的目录的webpack.config.js来作为配置文件
如果想指定另外的配置文件 需要执行
webpack -config webpack.custom.config.js

webpack要配置文件 webpack.config.js

安装加载器

任何静态资源都可以视作模块,然后模块之间可以项目依赖,通过webpack对模块进行处理后,可以打包成我们想要的静态资源
不同的文件需要不同的加载器.样式文件 css-loader sass-loader
第一步使用npm安装必须的loader
$ npm install style-loader css-loader url-loader babel-loader sass-loader file-loader jsx-loader --save-dev

gulp知识点

什么是gulp

Gulp是一个构建系统,执行开发中的常见任务.Gulp是基于Node.js来构建的.

1.全局安装gulp

$ npm install --global gulp

2.作为项目的开发依赖安装 在对应项目中安装

$ npm install --save-dev gulp

3 项目初始化 npm init 来 创建 package.json文件

$ npm init

通过初始化来设置项目的一些信息

gulp 获取文件

  • js/app.js 精确匹配文件
  • js/*.js 仅匹配js目录下的所有后缀为.js的文件
  • js/*/.js 匹配js目录及其子目录下所有后缀为js的文件
  • !js/app.js 从匹配结果中排除js/app.js,这种方法在你想要匹配特殊文件之外的所有文件非常管用
  • *.+(js|css) 匹配根目录下所有后缀为.js或者.css的文件

gulp 的API

gulp.task() gulp.src() gulp.dest() gulp.watch()

gulp.task(name, fn)这个你应经见过了

gulp.run(tasks…)尽可能多的并行运行多个task

gulp.watch(glob, fn)当glob内容发生改变时，执行fn

gulp.src(glob)返回一个可读的stream

gulp.dest(glob)返回一个可写的stream

gulp中的顺序执行方案

在gulp对任务的定义中,可以声明任务之间的依赖. 比如
gulp.task(“sass”,[“clean”],funtion(){
//do something
})
gulp.task(“clean”,function(){
return gulp.src(‘sample.css’)
.pipe(clean())
})
在声明依赖后,可以保证任务的顺序执行. 但是要记得 return 返回流

gulp plugins

gulp-autoprefixer

根据设置浏览器版本自动处理浏览器前缀.不必考虑各浏览器兼容前缀
github地址
npm地址
安装

npm install --save-dev  gulp-autoprefixer

使用

var autoprefixer=require("gulp-autoprefixer");

gulp.task("autopre",function(){
  gulp.src('app/css/**/*.css')
          .pipe(autoprefixer({
            browsers:["last 2 versions",">5%"],
            remove:true, // 是否去掉不必要的前缀
            cascade:true// 是否美化属性值 默认为true
            }))
            .pipe(gulp.dest("dist/css"))
  })

gulp-autoprefixer的browers参数的属性值 详细解释

  • last 2 versions: 主流浏览器的最新两个版本
  • last 1 Chrome versions: 谷歌浏览器的最新版本
  • last 2 Explorer versions: IE的最新两个版本
  • last 3 Safari versions: 苹果浏览器最新三个版本
  • Firefox >= 20: 火狐浏览器的版本大于或等于20
  • iOS 7: IOS7版本
  • Firefox ESR: 最新ESR版本的火狐

  • 5%: 全球统计有超过5%的使用率

gulp-changed

来过滤变动的文件

var gulp = require('gulp');
var changed = require('gulp-changed');
var ngmin = require('gulp-ngmin'); // just as an example 

var SRC = 'src/*.js';
var DEST = 'dist';

gulp.task('default', function () {
 return gulp.src(SRC)
    .pipe(changed(DEST))
    // ngmin will only get the files that 
    // changed since the last time it was run 
    .pipe(ngmin())
    .pipe(gulp.dest(DEST));
  })
gulp-plumber

此插件用来捕获处理任务中的错误
在gulp的挂到流任务处理中,如果摸个环节出了错,会导致整个任务中断,此可以捕获
安装

npm install --save-dev gulp-plumber

使用

var  plumber=require("gulp-plumber");
var minifyCss=require("gulp-minify-css");
gulp.task("minifycss",function(){
  gulp.src("app/css/**/*.css")
         .pipe(plumber())
         .pipe(minifyCss())
         .pipe(gulp.dest("dist/css"));
  })
