webpack 介绍

webpack 是前端一个工具,可以让各个模块进行加载, 预处理,再进行打包,它有grunt和gulp的所有的基本功能.

.支持CommonJs和AMD模块，意思也就是我们基本可以无痛迁移旧项目。
.支持模块加载器和插件机制，可对模块灵活定制。特别是我最爱的babel-loader，有效支持ES6。
.可以通过配置，打包成多个文件。有效利用浏览器的缓存功能提升性能。
.将样式文件和图片等静态资源也可视为模块进行打包。配合loader加载器，可以支持sass，less等CSS预处理器。
.内置有source map，即使打包在一起依旧方便调试。
.支持对css 图片等资源进行打包,从而无需借助Grunt或gulp

webpack的目标就是对项目中的静态资源进行统一整理,为产品的最终发布提供最优的打包部署方案.

安装webpack

两种方式安装webpack 一种单独下载webpack文件 另一种是用npm包的形式来安装(建议)
在全局中安装webpack
npm install webpack -g
安装完成之后,就有了全局的webpack命令. 直接执行此命令会默认使用当前的目录的webpack.config.js来作为配置文件
如果想指定另外的配置文件 需要执行
webpack -config webpack.custom.config.js

安装加载器

任何静态资源都可以视作模块,然后模块之间可以项目依赖,通过webpack对模块进行处理后,可以打包成我们想要的静态资源
不同的文件需要不同的加载器.样式文件 css-loader sass-loader
第一步使用npm安装必须的loader
$ npm install style-loader css-loader url-loader babel-loader sass-loader file-loader jsx-loader --save-dev

打包成多个资源文件

考虑到公共模块的问题, 利用插件就可以只能提取公共部分,以提供我们浏览器的缓存使用
在webpack.config.js 添加下面的代码

  • entry: 指定打包的入口文件,每一个键值对,就是一个入口文件
  • output: 配置打包结果,path定义了输出的文件夹,filename则定义了打包结果文件的名称,filename里面的
    [name]由entry中的键替换
  • resolve: 定义了解析模块路径时的配置,常用的就是extensions,可以用来指定模块的后缀,这样在引入模块时,
    就不需要写后缀了,会自动补全.
  • module: 定义了对模块的处理逻辑,这里可以用loaders定义了一系列的加载器,以及一些正则.当需要加载的文件匹配
    test的正则时,就会调用后面的loader对文件的处理.这正是webpack强大的原因.比如这里定义了凡是.js结尾的
    文件都是用babel-loader做处理,而 .jsx结尾的文件会先经过jsx-loader 处理,然后竞夺babel-loader来处理.
  • plugins: 在这里定义了需要使用的插件,比如commonsPlugin在打包多个入口文件时会提取公共的部分生成common.js

图片的打包

webpack对于静态资源来说,都是看做模块来加载的.图片也是作为模块来进行加载.图片是用url-loader加载的 图片在css文件中的url属性
其实是一种封装处理过的require操作.

上述两种方法都会对符合要求的图片进行处理。而要求就是在url-loader后面通过query参数的方式实现的，这里就是说只有不大于8kb的图片才会打包处理成Base64的图片

{test: /\.(jpg|png)$/, loader: "url?limit=8192"}

准备好以后,只需要在根目录下.执行webpack命令即可.
